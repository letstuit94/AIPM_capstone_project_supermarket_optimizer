## D2 — Recommendation candidates: design decision (LLM vs static table)

**Client asked:** can the recommendation candidate table be generated by an LLM from the user's needs?

**Our recommendation: a hybrid, LLM-*assisted* but grounded and deterministic-by-default.** Reasoning against a pure-LLM approach, and how to still use the LLM safely:

- **The app's core principle is "the LLM never invents nutrition facts" and "recommendations are deterministic and auditable."** A pure-LLM candidate generator breaks both: outputs vary run-to-run (fails the Gherkin acceptance tests, which assert fixed candidates) and the LLM could suggest foods whose nutrition it *imagines*.
- **But choosing *which food* to suggest is a recommendation, not a nutrition fact** — so an LLM *can* help, under four guardrails: every LLM-proposed food is (1) passed through the deterministic **exclusion filter** (diet/allergy/dislike), (2) **nutrition-resolved via OFF/BLS** (never the LLM), (3) **verified to actually move the target gap**, and (4) logged for auditability.

**MVP plan:**
1. **Ship the curated static candidate table below** (deterministic, testable, offline-safe) as the grounded baseline — this satisfies the acceptance criteria and works with zero LLM cost/latency.
2. **Add an optional "LLM candidate proposer" as a fast-follow** that expands/personalizes the pool (e.g. respecting behavioral context like "cooks rarely"), with the four guardrails. Feature-flag it so it can be A/B'd against the static table.

This keeps the safety/testability of the static table while capturing the personalization upside the client wants.

## D6 — Symptom → nutrient relevance & goal → nutrient relevance (full lookups)

Multipliers applied in BR-S4/S5 (default **1.0**; stack multiplicatively, cap 2.0). "Healthy/neutral" answers are 1.0 and omitted. Suppressors (<1.0) de-prioritize. **Heuristic — dietitian to confirm.**

### Symptom → nutrient
| Level-2 answer | Effect |
|---|---|
| Bowel < 3/week **or** stool = hard | fiber ×1.6, water reminder |
| Bloating = often/almost-daily **and** basket already high-fiber | fiber ×0.7 + "adjust meal composition" |
| Stomach = heartburn | (no nutrient change) surface reflux-friendly tip only |
| Stomach = nausea/pain (persistent) | no diagnosis → defer-to-professional note |
| Energy = afternoon crash | complex-carbs ×1.2, iron ×1.3 |
| Energy = tired most of day | iron ×1.4, B12 ×1.3, + sleep flag |
| Hunger = hungry most of day | protein ×1.4, fiber ×1.3 |
| Hunger = constantly hungry | protein ×1.5, fiber ×1.4 |
| Satiety = still hungry | protein ×1.3, fiber ×1.2 |
| Satiety = overly full | (no boost) portion tip |
| Sleep = poor | magnesium ×1.2, + caffeine-timing tip |
| Concentration = brain fog / mentally tired | iron ×1.3, omega3 ×1.2, B12 ×1.2 |
| Mood = low energy / stressed | magnesium ×1.2, omega3 ×1.2 |
| Hydration < 1 L | water reminder; if fiber high → fiber ×0.7 |
| Alcohol = weekly / several-times | global confidence ×0.85 + whole-food nudge |
| Supplement taken (multivitamin/iron/D/magnesium/omega-3/protein) | **suppress** the matching nutrient's gap priority ×0.6 (already supplementing) |

### Goal → nutrient
| Goal | Effect |
|---|---|
| Lose fat | protein ×1.4, fiber ×1.3 |
| Maintain | protein ×1.1 |
| Build muscle | protein ×1.5, carbs ×1.2 |
| Aggressive bulk | protein ×1.5, carbs ×1.3 |

## D7 — Allergen / intolerance → excluded-ingredient keywords (German + English)

Any candidate/recipe whose name or ingredients match a keyword for a user-selected allergen is excluded (case-insensitive, substring on normalized tokens). Lactose vs milk distinguished (lactose-free dairy is allowed for lactose intolerance but not milk allergy).

| Allergen | Exclude keywords |
|---|---|
| lactose | milch, sahne, rahm, molke, laktose (unless "laktosefrei") |
| milk | milch, sahne, butter, käse, joghurt, quark, rahm, molke, milk, cream, cheese, butter, yogurt |
| gluten | gluten, weizen, dinkel, gerste, roggen, malz, wheat, barley, rye (unless "glutenfrei") |
| wheat | weizen, weizenmehl, wheat, dinkel |
| eggs | ei, eier, egg, eigelb, eiweiß (as ingredient) |
| peanuts | erdnuss, erdnüsse, peanut |
| tree nuts | nuss, nüsse, mandel, haselnuss, walnuss, cashew, pistazie, pekan, macadamia, almond, hazelnut, walnut |
| soy | soja, tofu, edamame, sojasoße, soy |
| fish | fisch, lachs, thunfisch, hering, forelle, fish, salmon, tuna |
| crustacean shellfish | krabbe, garnele, shrimp, hummer, krebs, scampi, prawn, lobster, crab |
