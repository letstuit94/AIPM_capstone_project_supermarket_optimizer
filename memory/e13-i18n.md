Epic 13 (Internationalization DE/EN) implemented 2026-07-15. The frontend already had a custom i18n framework (`lib/i18n.tsx`: `t`, `LanguageProvider`, `useLanguage`, ~375-key catalog, header toggle, localStorage persistence). E13 closed the gaps.

**Decisions (confirmed with user):** the user chose **full backend localization** (not the "defer" or "light" options) ‚Äî so backend-generated prose is now DE/EN too, not just static UI.

**S1 gaps fixed:** default language now resolves stored-choice ‚Üí device locale (`navigator.language`, DE if starts with "de") ‚Üí EN (`detectDefaultLanguage` in i18n.tsx); previously hardcoded "en".

**S2 gap fixed:** AuthScreen was hardcoded English ‚Äî now uses `useLanguage` + `auth.*` keys.

**Full backend localization architecture:**
- `backend/app/services/i18n.py` (new) ‚Äî single catalog: `t(lang, key, **params)`, `nutrient(lang,key)`/`join_nutrients`, `action_verb`, `rationale_for(candidate,lang)` (picks `rationale_de`). All message templates in DE/EN.
- A `lang` param is threaded (default "en", so nothing breaks) from 3 endpoints ‚Äî `/nutrition/snapshot`, `/next-cart`, `/recommendations` (query `?lang=`) ‚Äî down through every prose builder: gap_detector, absolute_gap_detector, nutrition_model (build_dimension_snapshots + `disclaimer(lang)`), health_score, explainer, exclusion_filter (check_candidate), source_labels, progress_tracker, next_cart_engine, recommender (recommend_next_cart + find_pantry_match), easy_swaps, recipe_suggester, conflict_detector, nutrition_snapshot (build_snapshot_from_db ‚Äî lang is also in its cache key), and nutri_coach `_fallback_message` (the LLM path was already language-aware; the coach now uses the request `lang`, not profile.language, so it follows the header toggle).
- Data JSON: `recommendations.json` got 26 `rationale_de`; `recipes.json` got 44 `title_de`/`description_de` (done by two subagents, verified valid).
- Frontend: `api.ts` sends `?lang=` (from `getStoredLanguage`) on snapshot/next-cart/recommendations; `nutrientLabel(key,lang)` added to i18n.tsx and used in AnalysisCard bars + ResultsStep (gaps/dimensions/deltas/swaps); grouping tiers were already localized. ResultsStep re-fetches when `language` changes so backend prose updates on toggle.

**Known small edge (acceptable):** the fallback-category *value* in `source_labels` ("Estimated from category: vegetable") keeps the English category key; the composed OFF/BLS `data_source` bridge string stays English. Both are low-visibility (review/mapping). profile.language is no longer synced on toggle ‚Äî not needed since the request `lang` drives all backend output.
